<B><font face="Times New Roman" color="#ff9900" size="5">How the Web Works</font></B>
<p>The web is a bunch of computers that communicate with each other. When a person goes
to a web page like www.google.com, their computer sends a <em>HTTP Request</em> to a <em>server</em>. 
The server finds the appropriate HTML document and sends it back to the user's 
computer where a <em>web browser</em> interprets the page and displays it on the user's 
screen. <a href=”www.tutorialspoint.com/html/html_font_tag.htm“/html”> This Video<a/> does a good job of explaining.</p>
 
<B><font face="Times New Roman" color="#ff9900" size="5">HTML</font></B>
<p>HTML stands for <em>Hypertext Markup Language</em>. HTML documents form the majority of the content on the web. 
HTML documents contain <em>text content</em> which describes "what you see" and <em>markup</em> which describes "how it looks". 
<a href=”www.tutorialspoint.com/html/html_font_tag.htm“/html”> This Video<a/> gives a good overview.</p>
 
<B><font face="Times New Roman" color="#ff9900" size="5">Tags and Elements</font></B>
<p>HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each <b>element</b> by using HTML <b>tags</b>.   
<a href=”www.tutorialspoint.com/html/html_font_tag.htm“/html”> This Video<a/> explains the distinction well.</p>

<B><font face="Times New Roman" color="#ff9900" size="5">Why Computers are Stupid</font></B>
<p>Computers are stupid because they interpret instructions literally. This makes them very unforgiving since a small mistake
by a programmer can cause huge problems in a program.</p>

<B><font face="Times New Roman" color="#ff9900" size="5">Inline vs Block Elements</font></B>
<p>HTML elements are either <b>inline</b> or  <b>block</b>. Block elements form an "invisible box" around the content inside of them.</p>

About CSS Base

It's a common practice to apply CSS to a page that styles elements such that they are consistent across all browsers. We offer two of the most popular choices: normalize.css and a reset. Or, chose Neither and nothing will be applied.

     

Vendor Prefixing

About Vendor Prefixing

To get the best cross-browser support, it is a common practice to apply vendor prefixes to CSS properties and values that require them to work. For instance -webkit- or -moz-.

We offer two popular choices: Autoprefixer (which processes your CSS server-side) and -prefix-free (which applies prefixes via a script, client-side).

About External Resources
